# Code Style & Conventions

## General
*   **Commits:** Clear, concise, "Why" over "What".
*   **Documentation:** Focus on architectural decisions and complex logic.

## Go (Backend)
*   **Formatting:** Standard `gofmt`.
*   **Linting:** `golangci-lint` (default set).
*   **Error Handling:** Explicit error checking; wrap errors with context.
*   **Structure:** Follow Standard Go Project Layout.

## TypeScript (Frontend)
*   **Formatting:** `prettier`.
*   **Linting:** `eslint` (Vue recommended config).
*   **Style:** Composition API for Vue components.
*   **Types:** Strict type checking enabled.
